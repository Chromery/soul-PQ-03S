# New UI — indice della ricerca

Questa cartella raccoglie la fase di ricerca per il restyling di Soul Prospect Qualifier. Non contiene ancora la nuova implementazione dell'applicazione: definisce problemi, principi, priorità e direzioni visuali da validare prima di intervenire sui componenti di produzione.

## Deliverable

- [Review UX/UI](./ux-ui-review.md)
- [Direzione visuale](./art-direction.md)
- [Backlog proposto](./task-backlog.md)
- [Registro dei prompt visuali](./concept-prompts.md)
- Gallery web: `/new-ui/` nel deploy della branch

Gli asset visuali sono pubblicati sotto `apps/web/public/new-ui/` e comprendono:

- 5 screenshot dello stato attuale più una vista mobile;
- 3 direzioni per la schermata studi/dashboard;
- 3 varianti di onboarding sulla stessa schermata;
- 5 direzioni alternative per l'editor planimetrico.

## Decisione richiesta prima dell'implementazione

Scegliere una direzione primaria per il prodotto e una per l'editor. La raccomandazione iniziale è usare il concept **Quiet precision** come linguaggio generale e l'editor **Focus canvas** come architettura operativa, integrando il percorso guidato del concept **Workflow rail** per i nuovi utenti.
